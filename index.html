一、数据库概念&安装

1、什么是数据库呢？

顾名思义，就是存储数据的仓库

我们不需要去关注数据的存储格式，我们需要关注的是如何对存储的数据进行增删改查操作

学习重点：增删改查的SQL语句

Sql：

  • 添加：insert into
  • 修改：update
  • 删除：delete
  • 查询：select

2、数据库相关概念

1、数据库服务器：

​ 如果我们想写sql，必须先安装数据库服务器

分类：

  • 关系型数据库

    mysql、oracle、sqlserver、db2

  • 非关系型数据库

    redis、mongodb…

学习：mysql数据库

安装mysql服务器、卸载(见视频)

安装后如何登陆：

  • 搜索mysql，点进去直接输入密码，如果显示welecome…，则代表安装且登陆成功

  • 点击cmd进去，输入

    mysql -uroot -p1234

    所有人的用户名都是root

    密码：安装时设置的密码

2、连接数据库的图形化界面

常用的两种：

  • sqlyog
  • navicat

安装

3、数据库&数据表概念

数据库：相当于文件夹的概念

开发一个小米商城：为小米商城创建一个叫xiaomi 数据库，商城中涉及的用户数据、商品数据、订单数据…都可以存在 该数据库中

开发医疗系统：

​ 为医疗系统也创建一个数据库：xx

数据表：相当于文件的概念，用来直接存储数据

一个数据库对应多张数据表

问：数据表存储数据的形式是什么？？？？

​ 数据是由行和列组成 ，和excel表格数据形式一样

二、SQL

分类：

  • DDL
  • DML(2成)
  • DQL(8成)
  • DCL(不用)

1、DDL

特点：针对的是结构上的变化

1、操作数据库的SQL

  • 创建数据库：

格式：create database 数据库名;

  • 选中/切换数据库

格式：use 数据库名;

  • 删除数据库

格式：drop database 数据库名;

2、操作数据表的SQL

  • 创建数据表

格式：

​ create table 表名(

​ 字段1 类型(长度),

   字段2  类型(长度),

)

  • 删除数据表

格式：drop table 表名;

  • 查看表结构

格式：desc 表名;

2、DML

特点：结构不变，但是数据发生了变化

1、添加操作

格式：insert into 表名(字段1,字段2…) values(值1，值2…)

结论：

  • 插入字段的个数必须和值的个数匹配
  • 插入字段的顺序必须和值的顺序匹配

2、修改操作

格式：update 表名 set 字段1=值1,字段2=值2

全部修改：将所有行的数据的sname修改为了”小白龙”

全部修改用的比较少

条件修改：

​ update 表名 set 字段1=值1,字段2=值2 where 条件

修改时注意：

  • 确定要修改的字段有哪些
  • 确定修改的条件是什么

3、删除操作

删除所有数据：

​ 格式：delete from 表名；

根据条件删除：

​ 格式：delete from 表名 where 条件；

问题：truncate和delete的区别

delete：

​ 只是将表中所有的数据给删掉了，但是表结构还在

truncate：

​ 将整张表删除，然后重新创建一张结构一样的新表 表结构改变了

3、DQL

特点：结构不变，数据也不变

只是一个查询的操作

查询所有数据：

​ 格式：select * from 表名;

​ *：查询所有字段

查询指定的字段：

别名查询：

格式： 字段 as 别名 ，as 可以省略

根据条件查询：

格式：select * from 表名 where 条件;

练习题：

知识点：

  • and： 并且，需要同时满足

  • between A and B 求某一个区间的值 >=A <=B

  • or： 或者

  • in ： 后边跟集合 满足集合中任一条件即可

  • 模糊查询 关键字：like

    %： 多个任意字符

    like ‘李%’、like ‘%精’ 、 ‘%红楼%’

    _：一个占位符

三、其它查询

1、聚合查询

针对列去查询

  • count() 查询记录个数

    不统计值为null的信息

  • max() 查询最大值

  • min() 查询最小值

  • sum() 查询和

  • avg() 查询平均数

2、排序

关键字：order by 字段 asc/desc

分类：

  • 升序 asc 默认是升序
  • 降序 desc

3、分组

关键字：group by 字段

4、分页

关键字： limit 参数1，参数2

参数1：起始索引 从0开始

参数2：每页显示的条数 (这个值一般是固定的)

第一个计算公式：

​ 起始索引 = (当前页 - 1) * 每页显示的条数(参数2)

问题：当前页的值从哪来？？

​ 从页面上点击分页条时，将当前页值传过来

第二个计算公式：

​ 总页数 = 总记录数 % 每页显示的条数 == 0 ? (总记录数 / 每页显示的条数)

​ ： (总记录数 / 每页显示的条数 + 1)

例子：9 / 3 = 3页 10/3 = 3页 + 1 = 4页

问题：总记录数如何求

​ select count(*) from 表名;

四、约束

作用：为了保证数据的正确性、有效性、完整性

需要给字段去加约束

1、主键约束

关键字：primary key

特点：唯一且不能为空

作用：唯一的去标识某一条数据

自增策略：

​ 关键字：AUTO_INCREMENT 无需我们手动的管理主键的值

2、唯一约束

关键字：unique

特点：唯一(值不能重复) 但是可以为空

主键在表中只能有一个，但是唯一约束可以有多个

3、非空约束

关键字：not null
